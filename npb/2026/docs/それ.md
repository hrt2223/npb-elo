# NPB 2026 コード全体の流れと関係性

このフォルダは、NPB 2026年シーズンの試合結果を取得し、Eloレーティング、当日の勝率予測、順位表、スタメン情報を生成し、静的サイトとして公開するための一式です。

中心になる流れは次の通りです。

```text
NPB公式サイト
  ↓
fetch_results_2026.py
  ↓
game_results_jp_2026.csv
  ↓
update_daily.py  ← elo.py / elo_settings.py
  ↓
output/elo_*.csv
  ↓
make_elo_tables.py / make_elo_graphs.py
  ↓
output/elo_table.csv / output/elo_graph.png

NPB公式サイト
  ↓
update_today_probabilities.py  ← output/elo_final_ranking.csv / elo.expected_score()
  ↓
output/today_probabilities.csv
  ↓
fetch_lineups_2026.py
  ↓
output/today_lineups.csv

NPB公式サイト
  ↓
fetch_standings_2026.py
  ↓
output/standings.csv

output/*.csv / output/*.png
  ↓
make_site.py → site_builder/*
  ↓
site/index.html / site/{central,pacific}/*.html
```

## 1. データ取得

### `fetch_results_2026.py`

試合結果CSV `game_results_jp_2026.csv` を作る入口です。

- NPB公式サイトの月別日程ページと当日速報ページを取得する
- HTMLを `HTMLParser` でテキストトークン化する
- 終了済みの試合だけを `ResultGame` として抽出する
- 1試合をホーム行・ビジター行の2行にしてCSVへ書く
- `--merge-existing` を付けると既存CSVに新規取得分をマージする

このCSVはElo計算の最重要入力です。

### `fetch_standings_2026.py`

公式順位表を取得して `output/standings.csv` を作ります。

- セ・リーグ、パ・リーグそれぞれの順位表ページを読む
- チーム、試合数、勝敗、勝率、ゲーム差を抽出する
- サイトの順位表表示で使う

### `update_today_probabilities.py`

今日の試合予定とElo勝率予測を作ります。

- 月別日程ページから指定日の試合予定を取得する
- `output/elo_final_ranking.csv` から現在のEloを読む
- `elo.expected_score()` を使ってホーム勝率・ビジター勝率を計算する
- 公式の予告先発ページから予告先発を取得する
- `output/today_probabilities.csv` に保存する

ここで使うEloは、すでに `update_daily.py` で計算済みの最新ランキングです。

### `fetch_lineups_2026.py`

当日のスタメンを取得して `output/today_lineups.csv` を作ります。

- `output/today_probabilities.csv` を読んで当日の試合一覧を知る
- 試合開始60分前以降だけ取得対象にする
- 当日ページ・月別ページからスコアページURLを探す
- 各試合の `box.html` からスタメン表を読む
- すでに18人分そろっている試合は再取得しない

つまり、試合予定は `update_today_probabilities.py` が作り、スタメン取得はその予定CSVを入力として動きます。

## 2. Elo計算

### `elo_settings.py`

Eloのパラメータを定義します。

- `INITIAL_RATING`: 初期レーティング
- `K_FACTOR`: 1試合あたりの変動幅
- `LOGISTIC_BASE`: 期待勝率式の底
- `LOGISTIC_SCALE`: レーティング差のスケール
- `HOME_ADVANTAGE`: ホーム補正

`elo.py` と `update_today_probabilities.py` の両方が参照します。

### `elo.py`

Elo計算の中核です。

主な関数は次の通りです。

- `expected_score()`: 2チームのレーティングから期待勝率を計算する
- `actual_score()`: 勝ち=1、負け=0、引き分け=0.5 に変換する
- `update_ratings()`: 1試合分のレーティング更新を行う
- `load_games()`: `game_results_jp_2026.csv` を読み、試合単位のDataFrameに変換する
- `calculate_elo()`: 全試合を日付順に処理してランキング、試合別変化、履歴を作る
- `write_outputs()`: `output/` 配下へCSVを書き出す

`elo.py` は単体実行もできますが、通常運用では `update_daily.py` から呼ばれます。

### `update_daily.py`

日次更新の中心です。

処理内容は次の通りです。

1. `game_results_jp_2026.csv` を `elo.load_games()` で読む
2. `elo.calculate_elo()` で全体Eloを計算する
3. `output/` に全体版のCSVを書き出す
4. セ・リーグ、パ・リーグだけの出力を `output/central/`, `output/pacific/` に作る
5. 交流戦期間の出力を `output/interleague/` に作る
6. `elo_latest.txt` と `logs/update.log` に概要を残す

出力される主なファイルは次の通りです。

| file | role |
| --- | --- |
| `games.csv` | Elo計算に使った試合一覧 |
| `elo_final_ranking.csv` | 最新Eloランキング |
| `elo_by_game.csv` | 試合ごとのElo変化 |
| `elo_history.csv` | 日付・チームごとのElo履歴 |
| `elo_latest.txt` | 更新日時、試合数、期間、ランキングのサマリ |

## 3. 表とグラフ生成

### `make_elo_tables.py`

`elo_history.csv` を横持ちの表に変換します。

- 入力: `output/*/elo_history.csv`
- 出力: `output/*/elo_table.csv`
- 行は日付、列はチーム、値はElo

対象は全体、セ・リーグ、パ・リーグ、交流戦の4種類です。

### `make_elo_graphs.py`

`elo_table.csv` から推移グラフPNGを作ります。

- 入力: `output/*/elo_table.csv`
- 出力: `output/*/elo_graph.png`
- matplotlibを使う
- 試合がない場合は `No games found` の画像を出す

## 4. サイト生成

### `make_site.py`

サイト生成の入口です。

実体は `site_builder.builder.main()` を呼ぶだけです。

### `site_builder/`

静的HTML生成を役割ごとに分けたパッケージです。

| file | role |
| --- | --- |
| `config.py` | パス、リーグ、チーム色、チームslug、表示セクションの設定 |
| `builder.py` | サイト生成の全体手順 |
| `payload.py` | 各タブに渡す表示データを組み立てる |
| `dashboard.py` | トップページHTMLを組み立てる |
| `tables.py` | CSV読み込み、HTMLテーブル化、最新情報テキスト読み込み |
| `schedule.py` | 今日の試合、勝率、予告先発、スタメンのHTML化 |
| `standings.py` | 順位表HTMLの生成 |
| `team_pages.py` | 球団別ページと直近10試合テーブルの生成 |

`builder.py` の処理はシンプルです。

1. `config.SECTIONS` を順に処理する
2. `payload.build_section_payload()` で各セクションの表示データを作る
3. `dashboard.build_html()` で `site/index.html` を書く
4. `team_pages.write_team_pages()` で球団別ページを書く

トップページは、全体・セ・リーグ・パ・リーグ・交流戦のタブ構成です。
球団別ページは `output/elo_by_game.csv` を読み、各チームの直近10試合を表示します。

## 5. 自動更新

### GitHub Actions: `.github/workflows/npb-elo-daily.yml`

本番更新の中心です。

スケジュール実行で次を行います。

1. Python 3.12をセットアップする
2. `requirements.txt` をインストールする
3. 当月分の試合結果を `fetch_results_2026.py --month <当月> --merge-existing` で更新する
4. `update_daily.py`
5. `make_elo_tables.py`
6. `make_elo_graphs.py`
7. `update_today_probabilities.py`
8. `fetch_lineups_2026.py`
9. `fetch_standings_2026.py`
10. `make_site.py`
11. `game_results_jp_2026.csv`, `output/`, `site/` に差分があれば自動コミット・pushする

GitHub Actionsが動くため、ローカルPCが起動していなくてもサイトは更新できます。

### `daily_update.ps1`

ローカルPCで同じ更新をまとめて実行するPowerShellスクリプトです。

GitHub Actionsとの違いは、試合結果取得が `--today-only --merge-existing` になっている点です。
ログは `logs/daily_task.log` に残ります。

### `sync_from_github.ps1`

GitHub Actionsで作られた生成物をローカルPCへ取り込むための同期スクリプトです。

同期対象は次だけです。

- `npb/2026/game_results_jp_2026.csv`
- `npb/2026/output`
- `npb/2026/site`

Pythonコードや設定ファイルは同期対象にしていないため、ローカルで作業中のコードを上書きしにくい構成です。

### インストール系PowerShell

| file | role |
| --- | --- |
| `install_daily_task.ps1` | ローカル日次更新タスクを登録する |
| `install_sync_task.ps1` | GitHubからの同期タスクを登録する |
| `install_startup_sync.ps1` | タスク登録が難しい場合にログオン時同期を登録する |

## 6. 依存関係の見方

コード上の重要な依存関係は次の通りです。

```text
update_daily.py
  └─ elo.py
      └─ elo_settings.py

update_today_probabilities.py
  ├─ elo.expected_score()
  ├─ elo_settings.py の設定値
  └─ output/elo_final_ranking.csv

fetch_lineups_2026.py
  └─ output/today_probabilities.csv

make_elo_tables.py
  └─ output/*/elo_history.csv

make_elo_graphs.py
  └─ output/*/elo_table.csv

make_site.py
  └─ site_builder.builder
      ├─ site_builder.config
      ├─ site_builder.payload
      │   ├─ site_builder.tables
      │   ├─ site_builder.schedule
      │   ├─ site_builder.standings
      │   └─ site_builder.team_pages
      ├─ site_builder.dashboard
      └─ site_builder.team_pages
```

ポイントは、Eloの計算結果が後続処理の基準データになっていることです。

- `output/elo_final_ranking.csv` は今日の勝率計算に使う
- `output/elo_history.csv` は表生成に使う
- `output/elo_table.csv` はグラフとサイト表示に使う
- `output/elo_by_game.csv` は球団別ページの直近試合表示に使う
- `output/today_probabilities.csv` はスタメン取得とサイトの当日カード表示に使う
- `output/standings.csv` はサイトの順位表表示に使う

## 7. 読む順番のおすすめ

初めてコードを追うなら、次の順で読むと理解しやすいです。

1. `docs/PROJECT_STRUCTURE.md`
2. `daily_update.ps1` または `.github/workflows/npb-elo-daily.yml`
3. `update_daily.py`
4. `elo.py`
5. `update_today_probabilities.py`
6. `fetch_lineups_2026.py`
7. `make_site.py`
8. `site_builder/builder.py`
9. `site_builder/payload.py`
10. `site_builder/dashboard.py` と `site_builder/team_pages.py`

## 8. 注意点

- 生成物は `output/` と `site/` に集まる
- 本番更新はGitHub Actionsが担当する
- ローカルPCへの反映は `sync_from_github.ps1` が担当する
- Eloパラメータを変えたら、`update_daily.py` 以降の生成処理をすべて再実行する必要がある
- 予告先発やスタメンはElo計算の入力ではなく、当日カード表示用の補助データ
- 一部ファイル内の日本語文字列は文字化けして見える箇所があるが、構造としては「取得 → CSV → Elo/予測/順位 → HTML生成」という流れになっている
